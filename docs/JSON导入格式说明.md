# 数据导入JSON格式说明

## 概述

本系统支持通过JSON格式批量导入账户和账单数据。导入入口位于 **账户页面 → 导入按钮**。

---

## 一、账户导入

### 接口
```
POST /api/v1/import/accounts
```

### 请求格式
```json
{
  "accounts": [
    {
      "name": "账户名称",
      "type": 1,
      "icon": "",
      "color": "",
      "initial_balance": 0
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `name` | string | ✅ | 账户名称，不可为空 |
| `type` | int | ❌ | 账户类型，默认1 |
| `icon` | string | ❌ | 图标标识 |
| `color` | string | ❌ | 颜色色值，如 `#FF6B6B` |
| `initial_balance` | float | ❌ | 初始余额，默认0 |

### 账户类型对应关系

| type值 | 类型 |
|--------|------|
| 1 | 现金 |
| 2 | 银行卡 |
| 3 | 信用卡 |
| 4 | 支付宝 |
| 5 | 微信 |
| 6 | 其他 |

### 去重规则
- 按 `name` 字段去重
- 名称相同已存在的账户会**跳过**（不报错）

### 完整示例
```json
{
  "accounts": [
    {
      "name": "建设银行储蓄卡",
      "type": 2,
      "color": "#FF6B6B",
      "initial_balance": 10000
    },
    {
      "name": "支付宝",
      "type": 4,
      "color": "#1677FF",
      "initial_balance": 5000
    },
    {
      "name": "微信支付",
      "type": 5,
      "color": "#07C160",
      "initial_balance": 2000
    }
  ]
}
```

## 二、账单导入

### 接口
```
POST /api/v1/import/bills
```

### 请求格式
```json
{
  "bills": [
    {
      "account": "账户名称",
      "category": "分类名称",
      "type": 1,
      "amount": 100.00,
      "date": "2026-04-17",
      "time": "12:30",
      "remark": "备注",
      "tag_ids": []
    }
  ]
}
```

### 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `account` | string | ✅ | 账户名称，必须已存在 |
| `category` | string | ❌ | 分类名称，支持模糊匹配 |
| `type` | int | ❌ | 类型，默认1（支出） |
| `amount` | float | ✅ | 金额，必须大于0 |
| `date` | string | ✅ | 日期，支持 `YYYY-MM-DD` 或 `YYYY/MM/DD` |
| `time` | string | ❌ | 时间，支持 `HH:MM` 或 `HH:MM:SS` |
| `remark` | string | ❌ | 备注 |
| `tag_ids` | int[] | ❌ | 标签ID列表 |

### 账单类型

| type值 | 类型 |
|--------|------|
| 1 | 支出 |
| 2 | 收入 |

### 匹配规则

1. **账户匹配**：按 `account` 名称精确匹配已存在的账户
2. **分类匹配**：按 `category` 名称匹配，支持模糊匹配
   - 精确匹配优先
   - 支出分类只匹配支出类型，收入分类只匹配收入类型
   - 如 "午餐" 会匹配 "餐饮 > 午餐" 子分类

### 日期/时间格式

支持的日期格式：
- `2026-04-17`
- `2026/04/17`

支持的时间格式：
- `12:30`
- `12:30:00`

### 完整示例
```json
{
  "bills": [
    {
      "account": "建设银行储蓄卡",
      "category": "午餐",
      "type": 1,
      "amount": 35.5,
      "date": "2026-04-17",
      "time": "12:30",
      "remark": "工作餐"
    },
    {
      "account": "支付宝",
      "category": "打车",
      "type": 1,
      "amount": 28.0,
      "date": "2026-04-17"
    },
    {
      "account": "建设银行储蓄卡",
      "category": "工资",
      "type": 2,
      "amount": 15000,
      "date": "2026-04-01",
      "remark": "4月工资"
    },
    {
      "account": "微信支付",
      "category": "电影",
      "type": 1,
      "amount": 45,
      "date": "2026-04-19",
      "remark": "周末电影"
    }
  ]
}
```

## 三、导入流程

1. **选择类型**：切换「导入账户」或「导入账单」标签页
2. **查看示例**：页面中提供了JSON格式示例，点击「复制示例」可快速复制
3. **粘贴JSON**：将准备好的JSON数据粘贴到文本框
4. **预览解析**：点击「预览解析」检查JSON格式是否正确
5. **确认导入**：预览无误后点击「确认导入」

---

## 四、错误处理

### 账户导入错误
- 缺少 `name` 字段 → 报错并跳过
- 名称已存在 → 自动跳过

### 账单导入错误
- 缺少 `account` → 报错并跳过
- 账户不存在 → 报错并跳过
- 分类不存在 → 报错并跳过
- 金额≤0 → 报错并跳过
- 日期格式错误 → 报错并跳过

## 五、系统预设分类参考

### 支出分类（type=1）
- 餐饮（早餐、午餐、晚餐、零食、饮料）
- 交通（公交、地铁、打车、加油、停车）
- 购物（日用品、衣物、数码、美妆）
- 居住（房租、水电、物业、网费）
- 娱乐（电影、游戏、旅行、运动）
- 医疗（门诊、药品、体检）
- 教育（书籍、课程、培训）
- 通讯（话费、会员）
- 人情（红包、礼物、请客）
- 其他

### 收入分类（type=2）
- 工资
- 兼职
- 理财
- 红包
- 退款
- 其他

---

## 六、注意事项

1. 导入账单前请确保对应的**账户已存在**
2. 如果分类名称匹配不到，系统会报错，可尝试使用更通用的名称
3. 导入时会**自动更新账户余额**
4. 建议先小批量测试，确认无误后再大批量导入
5. JSON格式必须是**严格的标准JSON**，注意逗号、引号等语法
